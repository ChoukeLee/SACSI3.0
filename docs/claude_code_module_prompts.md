# Claude Code 后续模块开发提示词

> 使用方式：每次只复制一个模块提示词给 Claude Code。要求它先读取 `spec_v4.md`、`11_apartment_flow_framework.md`、`README.md`、`supabase/migrations` 和当前代码，再实施对应模块。

## 通用前置提示词

```text
请先阅读项目根目录的 spec_v4.md、11_apartment_flow_framework.md、README.md，以及 supabase/migrations 下的数据库迁移文件。

项目是科建地产房屋管理系统 SACIS 3.0，技术栈为 Next.js App Router + TypeScript + Tailwind CSS + Supabase PostgreSQL。首期业务只启用 SASCI11 / 11#公寓，但所有代码和数据库必须保留多楼栋扩展能力，不能把 11#写死为唯一业务边界。

开发要求：
1. 遵循现有 src/app、src/components、src/features、src/lib、src/types 结构。
2. 核心查询必须通过 building_id 或 unit_id 支持未来多楼栋筛选。
3. 当前 UI 已建立中法双语骨架。新增页面、按钮、表单、空状态、错误提示时，必须同时维护 `src/lib/i18n.ts` 的 `zh` 和 `fr` 文案，不能在组件里硬编码单语言业务文案。
4. 所有金额支持 XOF/CNY，报表默认换算为 XOF。
5. 权限先按管理员/老板/财务/前台预留，不要破坏后续 RLS 设计。
6. 完成后运行 typecheck/build，说明改动和剩余待办。
```

## 模块 1：Supabase 连接与数据访问层

```text
请完善 Supabase 数据访问层。

目标：
1. 基于现有 src/lib/supabase/client.ts 和 server.ts，创建统一的 repository/query 层。
2. 添加楼栋、房源、客户、日租、长租、出售、收款、财务流水的基础查询函数。
3. 查询函数默认支持 buildingId/buildingCode 参数，首期默认 SASCI11，但不能写死为唯一楼栋。
4. 添加数据库类型定义入口，可先手写基础类型，后续再由 Supabase CLI 生成替换。
5. 页面先可以继续使用 mock 数据，但要预留切换到真实查询的位置。

验收：
- npm run typecheck 通过。
- 代码中有清晰的数据访问边界，页面不直接散落 Supabase 查询。
```

## 模块 4：日租业务闭环

```text
请实现 11#日租业务闭环。

背景：
日租仅限 11#固定 21间房，但必须通过 unit_business_flags 中 business_type = daily_rental 获取，默认价格 40,000 XOF。

目标：
1. /daily-rentals 页面实现日历/列表视图，显示房号、可订/已订/维修/预订单、价格、客人姓名缩写。
2. 创建预订单，状态 pending_review，管理员确认后变为 confirmed。
3. 入住时将订单变为 checked_in，房态变为 daily_occupied。
4. 收款规则：至少预付一部分，不允许离店后付款。
5. 退房时一键计算金额、生成 payment、ledger_entry，房态变为 cleaning_pending。
6. 自动生成 cleaning_tasks；保洁完成后房态恢复 available。
7. 修改退房日期支持续住。
8. 检测长租合同占用冲突并阻止重叠预订。
9. OTA 双向同步只做接口占位和数据字段，不实现真实第三方对接。

验收：
- 日租完整流程可从页面走通。
- 日期冲突检测正确。
- 退房后有保洁任务和财务流水。
- npm run typecheck/build 通过。
```

## 模块 5：长租合同、应收与退租

```text
请实现长租业务模块。

目标：
1. /leases 页面实现合同列表、新建合同、合同详情。
2. 合同字段按 spec_v4.md：合同编号、起租日期、预计退租、实际退租、支付周期、付款日、押金、免租期、签约人/代理人、附件、状态。
3. 一个房间同一时间只能有一份 active 长租合同，前端和数据库错误都要友好提示。
4. 租金应收手动创建，不做自动生成。
5. 收款必须全额付清，不支持部分收款；支持提前收未来月份。
6. 逾期提醒提前 7 天，先写 notifications 记录。
7. 退租结算：检查未付租金、水电费已结清标记、押金扣除和退还，生成退租结算单数据。
8. 合同到期或退租后房态恢复 available。

验收：
- 长租签约、收款、退租流程可用。
- 写入 payments、ledger_entries、audit_logs。
- npm run typecheck/build 通过。
```
